# renamebin

`renamebin` は、非ASCII文字を含む `.png` ファイルを検索して安全なASCIIファイル名にリネームし、バイナリファイル内のファイル名参照を更新（パッチ適用）するDenoユーティリティです。

このツールは、ゲームのMOD制作などで、非ASCII文字を含むアセットファイル名をレガシーシステム向けに変更する必要があり、かつそのファイル名がShift_JISエンコーディングでバイナリファイルにハードコードされているような状況を想定して設計されています。

## 特徴

- **対象を絞ったリネーム:** 非ASCII文字を含む `.png` ファイルのみを特定してリネームします。
- **バイナリセーフ:** 新しいASCIIファイル名は、Shift_JISエンコーディングにおける元のファイル名のバイト長と一致するように `_` でパディング（文字埋め）されます。これにより、バイナリファイル内の文字列を置換してもファイル構造が破損しません。
- **インプレースなバイナリパッチ適用:** バイナリファイルをスキャンし、古いファイル名に一致する箇所をすべて新しいファイル名に置換します。
- **モジュール化されたスクリプト:** すべての処理を一度に行うメインスクリプトに加え、柔軟にステップバイステップで作業できる個別のスクリプトも提供します。

## 要件

- [Deno](https://deno.land/)

## 使い方

### クイックスタート（オールインワン）

以下のコマンドはすべてのステップを一度に実行します。新しいファイル名の生成、`.png` ファイルのリネーム、バイナリファイルへのパッチ適用を一括で行います。

```sh
deno run --allow-read --allow-write --allow-net https://code4fukui.github.io/renamebin/renamebin.js src.bin dst.bin textures
```

**引数:**
- `src.bin`: パッチを適用するソースバイナリファイル。
- `dst.bin`: 出力（パッチ済み）バイナリファイルのパス。
- `textures`: リネーム対象の `.png` ファイルが含まれるフォルダ。

### 高度な使い方（ステップバイステップ）

本ツールはモジュール化されたスクリプトで構成されており、個別に実行することで処理プロセスをより細かく制御できます。

**1. ファイル名マップの生成**

元の非ASCIIファイル名と新しいASCIIファイル名をマッピングするCSVファイルを作成します。

```sh
deno run --allow-read --allow-write --allow-net https://code4fukui.github.io/renamebin/maketxtfnlist.js textures map.csv
```
- `textures`: `.png` ファイルが含まれるフォルダ。
- `map.csv`: `fn`（元のファイル名）と `fn2`（新しいファイル名）のマッピングを出力するCSVファイル。

**2. 画像ファイルのリネーム**

生成されたマップに従って、ファイルシステム上の `.png` ファイルをリネームします。

```sh
deno run --allow-read --allow-write --allow-net https://code4fukui.github.io/renamebin/renametxtfn.js textures map.csv
```
- `textures`: `.png` ファイルが含まれるフォルダ。
- `map.csv`: 前のステップで作成したCSVマップファイル。

**3. バイナリファイルへのパッチ適用**

同じファイル名マップを使用してバイナリファイルを更新します。

```sh
deno run --allow-read --allow-write --allow-net https://code4fukui.github.io/renamebin/renamebinbymap.js src.bin dst.bin map.csv
```
- `src.bin`: ソースバイナリファイル。
- `dst.bin`: 出力バイナリファイルのパス。
- `map.csv`: CSVマップファイル。

## 仕組み

`renamebin.js` スクリプトは以下のプロセスを自動化します:

1.  **スキャン:** 指定されたフォルダ内を検索し、ファイル名に非ASCII文字を含む `.png` ファイルを検出します。
2.  **マッピング:** 検出された各ファイルに対して、一意の新しいASCIIファイル名（例: `0____.png`）を生成します。新しいファイル名は、**Shift_JIS**エンコーディングにおけるバイト長が元のファイル名のバイト長と完全に一致するようにアンダースコア（`_`）でパディングされます。これは安全にバイナリパッチを適用するために不可欠です。
3.  **リネーム:** ファイルシステム上の実際の `.png` ファイルをリネームします。
4.  **パッチ適用:** ソースバイナリを読み込み、Shift_JISエンコーディングされた元のファイル名が出現するすべての箇所を検索し、新しいファイル名に置換します。
